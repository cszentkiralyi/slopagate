### **What is slopagate.sh?**
It's a shell script designed to act as an interactive interface or "harness" for Ollama LLMs. It creates a command-line REPL (Read-Eval-Print Loop) where you can:
1.  Chat with the LLM (acting as an assistant).
2.  Trigger specific tools provided by the script (like `read`, `backup`, `list-directory`).
3.  Execute shell commands prefixed with `!`.

### **Configuration**
*   **Environment Variables:** You can set `SLOP_PORT`, `SLOP_HOST`, `SLOP_ENDPOINT`, and `SLOP_MODEL`.
*   **Config Files:** It looks for `.slop/SLOP.md` or `~/.config/slopagate/SLOP.md` to get default prompts/overwrites.

### **Interactions**
When you run the script (`./slopagate.sh`), you'll see a banner and an input prompt like `> `. Here's how you interact:

1.  **Standard Chat:**
    Just type your question or prompt, and it sends it to the configured Ollama model (defaulting to `qwen3.5:4b-32k`).
    *   Example: `Write a Python script that fetches data from an API.`

2.  **Run Shell Commands (`!` prefix):**
    Use `!` followed by the command to run it directly in your shell within the context of this session. The result is shown to you.
    *   Example: `!cat myfile.txt`
    *   Example: `!ls -la /tmp/`

3.  **Run Harness Commands (`/` prefix):**
    Use `/` followed by a command name to trigger specific features defined in the script's JSON configuration (currently implemented). The list of available tools is built-in.
    *   Example: `/read myfile.txt`
    *   Example: `/backup myfile.txt`

4.  **Clear Screen:**
    Use `clear` or `quit` commands.

### **Current Built-in Tools**
The script currently supports these functions (defined in the JSON payload):
*   `read`: Read file contents (supports optional line ranges).
*   `backup`: Back up a file (copies it with `.bak` extension).
*   `list-directory`: List contents of a directory.

### **How to Run It**
Assuming you have Ollama running locally:
```bash
./slopagate.sh
```

You might also need to ensure dependencies like `jq` and `curl` (which comes with `curl`) are installed, as the script relies on them heavily for parsing JSON and making HTTP requests.
